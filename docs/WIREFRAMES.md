# Persora — React UI/UX Sync Wireframes

> 작성일: 2026-05-31 · 갱신일: 2026-06-09 · 목적: `say-awsomely` 형제 앱의 React UI/UX 셸을 기준으로 Persora 화면을 재구성한다.
>
> 2026-06-09 변경: 새 페르소나 다이얼로그의 텍스트/캡처 이미지 토글을 제거하고, 대화 텍스트 + 카카오톡 대화 파일(.txt) 첨부 단일 흐름으로 교체. 분석하기 화면의 캡처 이미지 입력은 그대로 유지.

## 1. 공통 셸

동일하게 맞출 것:
- `say-awsomely`의 React 앱 셸 구조: sticky 상태바, 하단 탭바, 중앙 최대 폭, `bg-slate-50`, 흰색/반투명 바, 인디고 액센트.
- 상태바 우측: API 키 상태 버튼 + 언어 토글.
- 하단바: 3개 탭, 활성 탭 아이콘 scale-up, `fixed bottom-0`, `max-w-lg mx-auto`.
- 온보딩 모달, 토스트, 슬라이드업 다이얼로그, fade/slide 애니메이션 톤.

다르게 유지할 것:
- 앱 제목: `Persora` (한·영 공통, 단일 표기).
- logo/favicon: 현재 프로젝트의 `/app-logo.png`, `/favicon.png`, `/app-icon-192.png`, `/apple-touch-icon.png`.
- 핵심 탭 기능: `페르소나`, `분석하기`, `기록`.

```
┌──────────────────────────────────────────────┐
│  [app-logo] Persora             ● Gemini  한 │  ← sibling top status bar
├──────────────────────────────────────────────┤
│                                              │
│              active route content            │
│       max-width: 640px, mobile-first         │
│                                              │
├──────────────────────────────────────────────┤
│     페르소나        분석하기          기록    │  ← sibling bottom bar behavior
└──────────────────────────────────────────────┘
```

`say-awsomely`의 중앙 모달 패턴을 그대로 사용한다. 로고와 문구만 Persora에 맞춘다.

```
┌──────────────────────────────────────────────┐
│          dimmed backdrop + blur              │
│                                              │
│        ┌────────────────────────────┐        │
│        │          [app-logo]         │        │

│        │  [시작하기]                │        │

└──────────────────────────────────────────────┘
```

## 3. 탭 1 — 페르소나

형제 앱의 카드/빈 상태/버튼 밀도와 반경을 따른다. 화면 기능은 현재 앱의 상대 페르소나 목록/생성/상세를 유지한다.

```
┌──────────────────────────────────────────────┐
│ 페르소나                                     │
│ 상대방의 대화 기록으로 말투를 분석합니다     │
│                              [+ 새 페르소나] │
│                                              │
│ ┌──────────────────────────────────────────┐ │
│ │ 원형 아바타  김민준                      │ │
│ │             2026.05.31 · 요약 한 줄      │ │
│ │                                      ›   │ │
│ └──────────────────────────────────────────┘ │
│                                              │
│ empty: 중앙 아이콘 + 설명 + primary CTA      │
└──────────────────────────────────────────────┘
```

### 새 페르소나 다이얼로그

`PhraseAddDialog`와 같은 중앙 카드 모달/입력 스타일을 사용한다. 입력 방식 토글 없이 단일 흐름이다: 대화 텍스트를 직접 붙여넣거나, 카카오톡 대화 파일(.txt)을 첨부한다. 첨부 시 머리말 제거 + 말미(tail)만 잘라 textarea를 채우고, 사용한 글자수를 짧게 안내한다(사용자는 편집 가능).

```
┌─ 새 페르소나 만들기 ───────────────────────┐
│ 상대방 이름 [              ]  나의 이름 [ ]│
│ [📎 카카오톡 대화 파일(.txt) 첨부]          │
│  · 긴 파일은 최근 대화 위주로 잘라 사용     │
│ ┌────────────────────────────────────────┐ │
│ │ 대화 기록 textarea (붙여넣기/첨부 결과)  │ │
│ └────────────────────────────────────────┘ │
│  최근 N자 사용 (원본 M자)                   │
│                       [취소] [페르소나 생성]│
└────────────────────────────────────────────┘
```

### 상세 모달

형제 앱의 모달 배경/카드/액션 버튼 패턴을 사용한다. 내용은 기존 8개 항목 + 나/상대 탭을 유지한다.

## 4. 탭 2 — 분석하기

도메인 UI만 Persora 고유 화면으로 유지한다. 카드/textarea/primary button/결과 카드 스타일은 형제 앱과 동일한 Tailwind 토큰으로 구성한다.

```
┌──────────────────────────────────────────────┐
│ 분석하기                                     │
│ 페르소나 선택                                │
│ [김민준] [이지원] [박서연]                   │
│                                              │
│ 받은 메시지                                  │
│ ┌────────────────────────────────────────┐ │
│ │ 야 오늘 뭐해? 시간 돼?                 │ │
│ └────────────────────────────────────────┘ │
│                              [분석하기]     │
│                                              │
│ AI 심리 분석 카드                            │
│ 답변 후보 1~3 카드 + 복사 버튼               │
└──────────────────────────────────────────────┘
```

## 5. 탭 3 — 기록

형제 앱의 리스트 카드와 작은 보조 텍스트 스타일을 따른다. 펼치기/삭제 기능은 기존 기능을 유지한다.

```
┌──────────────────────────────────────────────┐
│ 기록                                         │
│ ┌──────────────────────────────────────────┐ │
│ │ 김민준 · 2026.05.31                     │ │
│ │ 야 오늘 뭐해? 시간 돼?                  │ │
│ │                                      ˅   │ │
│ │ expanded: 분석 / 후보 답변 / 삭제       │ │
│ └──────────────────────────────────────────┘ │
└──────────────────────────────────────────────┘
```

## 6. 구현 메모

- `src/App.tsx`는 형제 앱의 `App.tsx` 셸 구조를 따르고, 탭 정의만 Persora에 맞춘다.
- `Toast`, `ApiKeyStatus`, `LanguageToggle`, `OnboardingModal`은 형제 앱 컴포넌트 구조를 기준으로 포팅한다.
- 기존 vanilla DOM 기반 `components/*`, `routes/*`는 React 컴포넌트로 전환한다.
- 기존 도메인 레이어(`lib/persona.ts`, `lib/analysis.ts`, `repos/*`, `gemini.ts`, `i18n.ts`)는 최대한 유지하고 React에서 호출한다.
